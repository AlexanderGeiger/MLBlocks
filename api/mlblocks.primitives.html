mlblocks.primitives¶

Primitives module.

This module contains functions to load primitive annotations, as well as to configure how MLBlocks finds the primitives.

mlblocks.primitives.add_primitives_path(path)¶

Add a new path to look for primitives.

The new path will be inserted in the first place of the list, so any primitive found in this new folder will take precedence over any other primitive with the same name that existed in the system before.

Parameters:path (str) – path to add
Raises:ValueError – A ValueError will be raised if the path is not valid.
mlblocks.primitives.get_primitives_paths()¶

Get the list of folders where the primitives will be looked for.

This list will include the value of any entry_point named jsons_path published under the name mlprimitives.

An example of such an entry point would be:

entry_points = {
    'mlprimitives': [
        'jsons_path=some_module:SOME_VARIABLE'
    ]
}

where the module some_module contains a variable such as:

SOME_VARIABLE = os.path.join(os.path.dirname(__file__), 'jsons')
Returns:The list of folders.
Return type:list
mlblocks.primitives.load_primitive(name)¶

Locate and load the JSON annotation of the given primitive.

All the paths found in PRIMTIVE_PATHS will be scanned to find a JSON file with the given name, and as soon as a JSON with the given name is found it is returned.

Parameters:name (str) – name of the primitive to look for. The name should correspond to the primitive, not to the filename, as the .json extension will be added dynamically.
Returns:The content of the JSON annotation file loaded into a dict.
Return type:dict
Raises:ValueError – A ValueError will be raised if the primitive cannot be found.
